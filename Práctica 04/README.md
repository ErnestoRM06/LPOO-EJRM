Primero, las clases usadas fueron:

- Practica04: contiene el método main y las pruebas hechas.

- Aplicación: es la clase gestionadora de las demás.

- Usuario: es la clase padre, tiene métodos y atributos acerca de los datos del usuario

- Empleado: es clase hija de Usuario

- Administrador: es clase hija de Empleado

- Cliente: es clase hija de Usuario

En este caso la herencia se justifica para poder crear los usuarios y los distintos permisos que tienen. Un problema que note es que tal vez no le sea util el poder vender cosas a un administrador, pero lo deje porque tampoco creo que le estorbe. En otras  palabras, creo que la herencia tiene un ligero problema si quieres que una clase herede a otra, pero no quieres que herede todo de esa clase.
